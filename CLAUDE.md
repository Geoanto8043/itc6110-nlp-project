## Project overview

ITC6110 NLP group project. End-to-end NLP pipeline on the BBC News dataset (2,225 articles, 5 categories: sport, business, politics, tech, entertainment). Three sequential Jupyter notebooks cover every assessed step, feeding artefacts forward via `data/processed/`.

## Running the notebooks

Notebooks must be run **in order** — each one saves artefacts that the next loads.

```bash
# Launch Jupyter from the project root
jupyter notebook notebooks/01_data_features.ipynb
jupyter notebook notebooks/02_unsupervised_ml.ipynb
jupyter notebook notebooks/03_dl_rag_ui.ipynb
```

Run the Streamlit app (generated by NB3):
```bash
cd app
streamlit run app.py
```

Reload the BBC News dataset from scratch (if `data/raw/bbc_news.csv` is missing):
```python
from datasets import load_dataset
import pandas as pd
ds = load_dataset("SetFit/bbc-news")
df = pd.concat([pd.DataFrame(ds['train']).assign(split='train'),
                pd.DataFrame(ds['test']).assign(split='test')])
df.to_csv('data/raw/bbc_news.csv', index=False)
```

## Architecture

### Data flow

```
data/raw/bbc_news.csv
        │
        ▼ Notebook 1
data/processed/bbc_news_processed.csv   ← cleaned text + token lists
data/processed/tfidf_matrix.pkl         ← sparse (2225×10000) + vectoriser + labels
data/processed/doc_vectors_w2v.npy      ← (2225×100) mean Word2Vec embeddings
models/word2vec_bbc.model               ← gensim Skip-gram, size=100, window=5
        │
        ▼ Notebook 2
models/lda_model                        ← gensim LDA, K chosen by C_v coherence
data/processed/best_ml_pipeline.pkl    ← sklearn Pipeline (TF-IDF + best classifier)
data/processed/ml_results_summary.csv  ← accuracy/F1 for LR, NB, SVM
        │
        ▼ Notebook 3
models/lstm_classifier.pt              ← BiLSTM weights (PyTorch)
models/distilbert/                     ← HuggingFace checkpoint (fine-tuned)
data/processed/faiss_index.index       ← FAISS IndexFlatIP, 384-dim embeddings
data/processed/articles_for_rag.csv   ← text + category for all 2225 articles
app/app.py                             ← Streamlit UI (written by NB3 cell)
app/requirements.txt
```

### Notebook responsibilities

| Notebook | Steps | Key outputs |
|----------|-------|-------------|
| `01_data_features.ipynb` | 1–3: Data, Preprocessing, Embeddings | TF-IDF matrix, Word2Vec model, t-SNE figures |
| `02_unsupervised_ml.ipynb` | 4.1 + Task 1 ML + XAI | LDA model, pyLDAvis HTML, LR/NB/SVM pipelines, LIME figures |
| `03_dl_rag_ui.ipynb` | Task 1 DL + Task 2 RAG + Task 3 UI | BiLSTM, DistilBERT, FAISS index, Streamlit app |

### Design decisions worth knowing

- **DistilBERT uses original `text`**, not `text_processed`. BERT's subword tokeniser performs better on natural text; lemmatised tokens hurt it.
- **LSTM uses `text_processed`** (lemmatised tokens from NB1) via a custom vocab built from train split only.
- **TF-IDF in NB2 ML section is refit on train only** (inside a `Pipeline`) even though a full-corpus TF-IDF exists in `tfidf_matrix.pkl` — this prevents leakage.
- **RAG uses `IndexFlatIP`** (inner product) with L2-normalised embeddings, giving exact cosine similarity search. Appropriate for 2,225 vectors.
- **Flan-T5-base runs on CPU** for generation stability on MPS; sentence-transformers encoder runs on MPS.

## Environment

Python 3.12. No virtual environment is set up in the repo — packages are installed system-wide via `pip3.12 install`.

Key packages and known version constraints:
- `scikit-learn 1.8`: `n_iter` removed from `TSNE` (use `max_iter`); `multi_class` removed from `LogisticRegression`
- `transformers 5.x`: `eval_strategy` (not `evaluation_strategy`); `use_mps_device` removed — MPS auto-detected by `Trainer`
- `pyLDAvis 3.4`: use `pyLDAvis.gensim_models.prepare()` (not the old `pyLDAvis.gensim`)
- `scipy sparse`: boolean indexing requires `.values` to convert pandas Series to numpy array first

## GPU

Notebook 3 uses Apple Silicon MPS via PyTorch:
```python
device = torch.device('mps')   # set at top of NB3
```
NB1 and NB2 are CPU-only (sklearn, gensim, scipy have no MPS backends).

Corpus: 2,225 raw → **2,126 after dropping 99 duplicates**; splits are **1,194 train / 932 test**. Anywhere this repo says the processed corpus or TF-IDF matrix is 2,225 rows, it is wrong.

**Do not restate the DistilBERT/SVM gap as "the transformer won."** It is 0.21 points = 2 articles out of 932, and it flipped sign between two runs of the same seeded notebook. The real finding is that the two *deep* models are 15.7 points apart while DistilBERT and a bag-of-words SVM are 0.21 apart — pre-training explains the results, not depth.
